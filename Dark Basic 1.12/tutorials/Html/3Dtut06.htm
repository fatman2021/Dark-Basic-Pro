3D Tutorials: 1. The Basics
6. Texturing objects

Purpose:  To explain how to apply textures to objects.

    Rem Setup sync
      Sync On
      Sync Rate 30

    Rem Load Textures
      Load image "cottag02.bmp",1
      Load image "barry.bmp",2
 

    Rem Make Cubes and place randomly
      For x = 1 to 5
          Make object cube x,100
          Position object x,Rnd(2000),0,Rnd(2000)
          Set object collision to boxes x
          Texture object x,1
      Next x

    Rem Make sphere
      Make object sphere 10,50
      Position object 10,-100,0,-100
      Set object collision to spheres 10
      Texture object 10,2

    Rem Main loop
      Do
       Rem  Store Object angle Y in aY#
         aY# = Object angle Y(10)

       Rem  Control input for camera
         If Upkey()=1 then Move object 10,10

         If Leftkey()=1
             Yrotate object 10,Wrapvalue(aY#-5)
         Endif

         If Rightkey()=1
            Yrotate object 10,Wrapvalue(aY#+5)
         Endif

       Rem Detect collision
         If Object collision(10,0)>0
            Position object 10,X#,0,Z#
         Endif

       Rem get player object position and store in X# and Z#
         X# = Object position x(10)
         Z# = Object position z(10)

       Rem get new camera position and store in cZ# and cX#
         cZ# = Newzvalue(Z#,aY#-180,100)
         cX# = Newxvalue(X#,aY#-180,100)

       Rem position camera
         Position Camera cX#,75,cZ#

       Rem point the camera at the player object
         Point camera X#,25,Z#

       Rem Scroll textures on cubes
         Scroll object texture 1,0.005,0
         Scroll object texture 2,0,0.005

       Rem Scale textures on cubes
         Scale object texture 3,0.998,0.998
         Scale object texture 4,1.001,1.001

       Rem Refresh Screen
         Sync

      Loop

In this tutorial we will show you how to apply textures to your objects. Textures used in DarkBASIC are loaded from bit map image files with the ".bmp" extension. It's a good idea to create your textures with the dimensions divisible by two. Some video cards can't handle odd numbered dimensions or dimensions not of base two. This means the dimensions of your image can be 256x256, 32x128, 256x16 or any other size with a similar base. You can load images larger than 256x256 but check your hardware specifications to be sure your video card can handle huge textures.

        Load image "cottag02.bmp",1
        Load image "barry.bmp",2

The "Load Image" command instructs DarkBASIC to load an image from a bit map. The first parameter of the command is the file name of the bit map you wish to load. You must punctuate the file name with quotation marks or an error will occur. In this parameter, You can use directory names such as, "C:\Darkbasic\Images\mybitmap.bmp". If you are storing your images in a subdirectory of your project directory, you will have to specify the directory name when loading images. You only need to specify the subdirectory where the texture is located and not the full path to the image. This is usually a good idea if your program is using many media files. The second parameter is the number of the image. The number is like the number used when you create an object. It is a unique identifier for the image. No two images can have the same number.

        Texture object x,1

        Texture object 10,2

These two lines occur in separate places before your main loop. The first line is in the "For Next" loop we use to create our cube objects. The second line is after we create our player object. The "Texture Object" command is used to apply an image to an object. The first parameter is the number of the object we wish to apply the texture to. The second number is the number of the image we wish to apply to the object.

        Scroll object texture 1,0.005,0
        Scroll object texture 2,0,0.005

The "Scroll Object Texture" command moves a texture, that is applied to an object or polygon, across its surface up and down and left to right. The first parameter is the number of the object whose texture will be scrolled. The next two parameters are the U and V coordinates. You can think of these coordinates as the X and Y coordinates of the texture as you are viewing it in two dimensions. Placing a value other than zero in these parameters will scroll the texture on the object each time the command is called. You can use both positive and negative values depending on which direction you wish to move the texture on the object. We are using a very small floating point value in this example. The U and V scroll values can be thought of as a percentage of the dimensions of the textures. If we set the value to "0.5", which represents 50 percent, the texture would move half way accross the cube face, which is too fast.

        Scale object texture 3,0.998,0.998
        Scale object texture 4,1.001,1.001

The "Scale Object Texture" command is similar to the previously described command, except this command scales the texture on the object. The first parameter is the number of the object whose texture we wish to affect. The next two parameters are again, the U and V coordinates of the texture. Changing these two values to something other than zero will expand or shrink the size of the texture as it is tiled on the object. By setting the U and V values to a value of 1, no scaling will occur. Setting the values less than 1 will cause the texture to shrink on the object. Setting the value greater than 1 will cause the texture to expand on the object.

You should now know how to load an image from a file and apply it to an object as well as scale and scroll the texture on an object. Try some of the following changes to see how they affect the textured cubes.

Try changing the U and V values for the "Scroll Object Texture" command to see how they affect the scrolling speed of the textures on the objects.

Try changing the U and V values for the "Scale Object Texture" command to see how they affect the scale of the texture on the objects.
